# Designer-ally request — expand each civ's city list (11b, 2026-07-19)

From run-F item 11b: a >8-city empire currently runs out of named cities and falls
back to "New <name>" / "<Civ> Outpost <n>". We want each of the 14 civs' `cities`
list in `data/civs.json` extended from **8 (7 for Aztecs/Mongols) to ~16** real
historical names, so a large empire keeps getting authentic city names.

## What to deliver
For each civ below, **~8 ADDITIONAL city names** to APPEND after the current list
(target ~16 total). Keep the current names + order unchanged (the first is the
capital); just add more. Return per civ as a simple appended list.

## Writing constraints
- **Real historical/authentic city names** for that civilization — same flavor as the
  existing list (a mix of ancient + later-period names is fine; match what's already
  there).
- **Printable ASCII only** — game state stores printable-ASCII strings, so normalize
  any accents/diacritics to ASCII (e.g. `Zürich`→`Zurich`, `Kraków`→`Krakow`,
  `Xi'an`→`Xian`). Keep them recognizable.
- **No duplicates** within a civ, and avoid reusing a name already on ANOTHER civ's
  list (cross-civ collisions read oddly).
- Don't repeat any name already in that civ's current list (shown below).
- These are DATA (names only), keyed by civ — no prose, no effects.
