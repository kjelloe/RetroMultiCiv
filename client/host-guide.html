Host your own RetroMultiCiv server

One Node process serves the browser client and runs the authoritative game — no build step, one dependency (ws), no database, no secrets. Get Node, get the repo, run one command.

Quick start · Ubuntu + systemd · Docker · Public VM with TLS · Raspberry Pi · Flags reference

This is the in-browser summary. The full walkthrough — including the complete nginx/Let's Encrypt VM setup — lives in docs/how-to-host.md in the repository.

Quick start

Prerequisites: Node.js LTS (v18+, v22 recommended) and git.

git clone https://github.com/<owner>/multiciv.git
cd multiciv
npm ci          # installs ws; no build step
./run.sh        # http://localhost:8123

Open http://localhost:8123/client/?server=1 — the client joins the game over /ws. LAN players use http://<your-ip>:8123/client/?server=1 and the 5-letter join code from the lobby. On Windows, use .\run.ps1.

Without ?server=1 the client runs a fully local engine (single-player / hotseat) — no server needed. ?server=1 is what makes it a hosted game others can join.

Ubuntu + systemd

For a box that keeps running across reboots. Install Node LTS, clone into /opt/multiciv, npm ci, then create /etc/systemd/system/multiciv.service:

[Unit]
Description=RetroMultiCiv server
After=network.target

[Service]
Type=simple
User=multiciv
WorkingDirectory=/opt/multiciv
ExecStart=/usr/bin/node server/index.js --port 8123 --host 127.0.0.1
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
sudo systemctl enable --now multiciv
journalctl -u multiciv -f       # watch it
# update: cd /opt/multiciv && git pull && npm ci && sudo systemctl restart multiciv

Use --host 127.0.0.1 behind a reverse proxy; drop it (default 0.0.0.0) for direct LAN access. Run as a dedicated non-login user, never root.

Docker

The repo is the app, so the image just copies source, installs ws, and runs Node. A Dockerfile and compose.yaml ship in the repo root.

docker build -t retromulticiv .
docker run --rm -p 8123:8123 retromulticiv
# flags reach the server: docker run … retromulticiv --civs 6 --size large
# persist games: -v multiciv-saves:/app/saves

Or docker compose up -d. A GitHub Actions workflow can publish the image to GHCR, but that stays off until the repo owner opts in (it's a public artifact channel).

Public VM with TLS

A single small VM (2 vCPU / 4 GB is ample) serves https://yourdomain with nginx terminating TLS in front of Node on 127.0.0.1:8123. Node is never exposed directly; the firewall opens only 80/443 (+ SSH). The one RetroMultiCiv-specific detail is the nginx WebSocket upgrade block for /ws — without it the game socket never connects:

map $http_upgrade $connection_upgrade { default upgrade; '' close; }

server {
    listen 80;
    server_name yourdomain.example;

    location /ws {
        proxy_pass http://127.0.0.1:8123;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_read_timeout 1h;
    }
    location / {
        proxy_pass http://127.0.0.1:8123;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Point an A record at the VM, then sudo certbot --nginx -d yourdomain.example --redirect rewrites the site to HTTPS. The WebSocket becomes WSS automatically — the client uses wss:// on an HTTPS page. TLS is required for a public host — seat tokens cross the game socket, so plain ws:// over the internet would expose them (a trusted LAN is fine). Full step-by-step (firewall, deploy user, systemd, securing saves/) is in docs/how-to-host.md.

Raspberry Pi

A Pi hosts a small game fine. Only the deltas from the Ubuntu path:

  • Node on ARM via NodeSource (setup_22.x) — same command.
  • To bind port 80 without nginx, grant the capability: sudo setcap 'cap_net_bind_service=+ep' "$(command -v node)" (re-run after each Node upgrade).
  • On 1–2 GB models keep the civ count modest — each civ adds AI work per turn. A medium map with ≤ 6 civs is comfortable on a Pi 4.

Server flags

FlagDefaultMeaning
--port N8123HTTP + WebSocket port.
--host IP0.0.0.0Bind address; 127.0.0.1 behind a proxy.
--seed NrandomWorld seed.
--civs N2Civilizations (2–14, capped by map size).
--size Smediumxsmall…huge.
--game FILE—Resume a saved server game.
--no-saveoffDisable autosave after each command.
--debugoffDev only — serves the whole repo.
Keep --debug off in production. The default is hardened: only /client/, /engine/, /shared/, and /data/ are served, so saves/ (seat tokens + game codes) never reaches the wire. saves/*.json is the only game state on disk — back that up.
